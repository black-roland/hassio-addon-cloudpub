## 2.3.2-1

В этом обновении — важные архитектурные изменения: вместо **локальной сборки контейнеров** теперь используются **контейнеры, собираемые предварительно** ([pre-built containers](https://developers.home-assistant.io/docs/add-ons/publishing)).

У такого подхода сразу несколько плюсов:
1. Быстрая установка аддона и его обновлений;
2. Повышенная надёжность (меньше зависимостей и точек отказа);
3. Не зависит от GitHub Container Registry, который может быть заблокирован;
4. Чуть меньший износ SD-карт на одноплатниках.

Теперь используется **надёжный реестр контейнеров** в российском облаке, что решает проблемы из-за интернет-блокировок:
- Невозможность загрузки базового Docker-образа с `ghcr.io`;
- Ошибки при локальной сборке контейнера из-за невозможности скачать клиент CloudPub с официального сайта.

Теперь аддон целиком скачивается из нового реестра с предустановленным клиентом и базовым образом.

Правда есть и минусы:

1. Гипотетически новый реестр тоже может быть заблокирован (маловероятно);
2. Хостинг нового реестра платный, но:
   - Стоимость минимальна из-за небольшого количества пользователей;
   - Расходы полностью покрываются благодаря вашим донатам! [Большое спасибо за поддержку](https://github.com/black-roland/hassio-addon-cloudpub?tab=readme-ov-file#%D0%BF%D0%BE%D0%B4%D0%B4%D0%B5%D1%80%D0%B6%D0%BA%D0%B0-%D0%B0%D0%B2%D1%82%D0%BE%D1%80%D0%B0)! 🤗

Ниже привычный список изменений.

### Добавлено

- **Отдельная canary-версия**: доступ к самым свежим, но непротестированным обновлениям клиента CloudPub. Canary устанавливается по-старому (с локальной сборкой).

### Изменено

- **Синхронизация версий**: версия аддона теперь совпадает с версией клиента CloudPub, чтобы избежать путаницу.
- **Оптимизация доставки**: ускоренное скачивание и установка благодаря новому реестру.
